# Задание: напиши клиент для игры «Остров бартера»

## Контекст

Ты подключаешься к уже работающему серверу экономической игры.
Игроки торгуют товарами (рыба, дерево, ткань, зерно, глина), у каждого —
случайный инвентарь и секретная цель. Твоя задача — написать веб-клиент,
через который удобно играть.

Сервер: `http://<адрес>:3000`  
Документация по всем эндпоинтам: смотри `README.md` в этом репозитории.

---

## Минимальные требования (нужно сделать обязательно)

1. **Форма входа** — поле для имени, кнопка «Войти».  
   Вызывает `POST /player/join`, сохраняет `playerId` и `token`.

2. **Инвентарь** — показывает, сколько каждого товара есть у игрока.  
   Данные: `GET /player/:id`

3. **Секретная цель** — что нужно собрать, чтобы выиграть.  
   Есть в том же ответе `/player/:id`, поле `goal`.

4. **Форма предложения сделки**:  
   — выбор другого игрока  
   — что отдаёшь (товар + количество)  
   — что просишь взамен  
   Вызывает `POST /trade/offer`

5. **Список входящих предложений** с кнопками «Принять» / «Отклонить».  
   Данные: `GET /trades/pending/:playerId`  
   Действия: `POST /trade/accept/:tradeId`, `POST /trade/decline/:tradeId`

6. **Список всех игроков** (кто уже выполнил цель, а кто нет).  
   Данные: `GET /game/state`

---

## Дополнительные задания (на выбор)

- **WebSocket** — получать уведомления о новых сделках в реальном времени,  
  не перезагружая страницу. Подключение: `ws://<адрес>:3000`,  
  первое сообщение: `{ "type": "auth", "token": "..." }`  
  События от сервера: `trade_offer`, `trade_accepted`, `trade_declined`,  
  `game_event`, `currency_enabled`, `round_started`

- **Ракушки (валюта)** — когда учитель включает валюту, у каждого  
  появляется баланс. Добавь возможность включать ракушки в сделку  
  (поля `shells` в `offer` и `request`).

- **Визуализация** — нарисуй инвентарь картинками или иконками вместо текста.

- **История сделок** — показывай принятые и отклонённые предложения.

- **Индикатор прогресса к цели** — сколько из нужных товаров уже есть.

- **Таймер** — секундомер с момента начала раунда.

---

## Технические ограничения

- Только **HTML + CSS + JavaScript** — никаких фреймворков, никаких сборщиков.
- Один файл `client.html` (или несколько, если хочешь разбить на части).
- Никаких библиотек кроме тех, что уже встроены в браузер (`fetch`, `WebSocket`).
- Файл можно открывать прямо с компьютера (двойной клик → браузер) —  
  сервер настроен так, что это работает.

## Подсказки

**Первым делом — укажи адрес сервера в начале файла:**
```js
// Замени на IP учителя, например: 'http://192.168.1.42:3000'
const SERVER = 'http://192.168.1.42:3000';
```
Дальше используй `SERVER` везде вместо захардкоженного адреса.

---

**Как сделать запрос с токеном:**
```js
const response = await fetch(SERVER + '/player/' + playerId, {
  headers: { 'x-player-token': token }
});
const data = await response.json();
```

**Как предложить сделку:**
```js
await fetch(SERVER + '/trade/offer', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'x-player-token': token
  },
  body: JSON.stringify({
    fromId:  playerId,
    toId:    'id-другого-игрока',
    offer:   { item: 'fish', qty: 2, shells: 0 },
    request: { item: 'wood', qty: 1, shells: 0 }
  })
});
```

**Как подключиться по WebSocket:**
```js
const ws = new WebSocket(SERVER.replace('http', 'ws'));

ws.onopen = () => {
  ws.send(JSON.stringify({ type: 'auth', token: token }));
};

ws.onmessage = (event) => {
  const msg = JSON.parse(event.data);
  console.log(msg.type, msg); // trade_offer, game_event, ...
};
```
